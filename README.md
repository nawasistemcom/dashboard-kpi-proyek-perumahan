# Dashboard KPI Proyek Perumahan

Dashboard KPI Proyek Perumahan adalah sistem kontrol proyek yang membantu developer, Project Manager, Site Manager, admin proyek, tim teknik, penjualan, dan estate membaca kondisi proyek dari satu sumber data.

Dalam proyek perumahan, data sering tersebar di banyak file. Progress pembangunan ada di laporan teknik, status kontraktor berada di file lain, data penjualan berjalan sendiri, sedangkan BAST dan komplain konsumen sering tidak langsung terhubung dengan laporan utama.

Karena itu, proyek membutuhkan dashboard yang mampu menyusun data menjadi tampilan kontrol yang lebih rapi, visual, dan mudah dibaca.

## Cara Kerja Dashboard

Prinsip kerja dashboard ini sederhana.

Petugas cukup mengisi **Sheet Input Data**.

Setelah data utama masuk, beberapa bagian lain dapat membaca data tersebut secara otomatis, seperti:

- Dashboard Proyek untuk tim teknik
- Dashboard Penjualan
- Sheet Estate BAST
- Master Control sebagai resume utama proyek

Dengan alur ini, tim tidak perlu mengisi banyak laporan secara berulang. Satu sumber data dapat membantu beberapa kebutuhan evaluasi proyek.

## Data yang Dapat Dipantau

Dashboard KPI Proyek dapat membantu membaca beberapa indikator penting, seperti:

- Progress pembangunan
- Deviasi schedule
- Kinerja kontraktor
- Quality control
- Komplain konsumen
- BAST atau serah terima unit
- Status penjualan
- Resume kondisi proyek
- Risiko proyek
- Rekomendasi tindak lanjut

## Manfaat untuk Project Manager

Dashboard ini membantu Project Manager membaca kondisi proyek lebih cepat.

Jika progress terlambat, tim dapat melihat sumber masalah lebih awal. Jika kontraktor tertinggal, Project Manager dapat melakukan evaluasi berdasarkan data. Selain itu, jika BAST mulai terlambat, tim dapat melihat hubungan antara pekerjaan lapangan, administrasi, dan status konsumen.

Dengan begitu, keputusan proyek tidak hanya berdasarkan perkiraan, tetapi berdasarkan data yang lebih jelas.
